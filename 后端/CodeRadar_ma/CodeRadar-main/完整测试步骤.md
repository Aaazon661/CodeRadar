
#### 4.2 准备测试文件


### 测试方式 2：使用 PowerShell 命令行

#### 4.4 测试文件上传 API

```powershell
# 设置变量
$uri = "http://localhost:8080/api/file/upload"
$filePath = "D:/test.java"
$userId = 1

# 创建表单数据
$form = @{
    userId = $userId
    file = Get-Item -Path $filePath
}

# 发送请求
$response = Invoke-RestMethod -Uri $uri -Method Post -Form $form

# 显示结果
$response | ConvertTo-Json -Depth 10
```

**预期返回结果：**
```json
{
  "resultId": 1,
  "requestId": "req-xxxxxxxxxx",
  "userId": 1,
  "fileId": 1,
  "model": "deepseek-chat",
  "reviewTime": "2024-01-15T10:30:00",
  "summary": "代码存在多个安全和质量问题...",
  "rawJson": "{...}",
  "createdAt": "2024-01-15T10:30:00"
}
```

#### 4.5 测试获取审查结果 API

```powershell
# 使用上一步返回的 requestId
$requestId = "req-xxxxxxxxxx"  # 替换为实际的 requestId
$uri = "http://localhost:8080/api/file/result/$requestId"

# 获取结果
$result = Invoke-RestMethod -Uri $uri -Method Get

# 显示结果
$result | ConvertTo-Json -Depth 10
```

## 🔍 第六步：常见问题排查

### 问题 1：无法访问 http://localhost:8080

**检查步骤：**
```powershell
# 检查端口是否被占用
netstat -ano | findstr :8080
```

**解决方案：**
- 确认 Spring Boot 已启动
- 检查防火墙设置
- 如果端口被占用，修改 application.yml 中的端口号

### 问题 2：上传文件返回 500 错误

**可能原因：**
1. 数据库连接失败
2. 表结构不存在
3. AI API Key 未配置或无效

**检查步骤：**
1. 查看 IntelliJ IDEA 控制台的错误日志
2. 检查数据库连接
3. 验证 API Key 是否正确

### 问题 3：AI 审查失败或超时

**可能原因：**
1. API Key 无效
2. 网络连接问题
3. API 配额用尽
4. 文件内容过大

**解决方案：**
1. 验证 API Key：访问 https://platform.deepseek.com/
2. 检查网络连接
3. 查看 API 使用配额
4. 减小测试文件大小

### 问题 4：数据库中文乱码

**解决方案：**
```sql
-- 检查数据库字符集
SHOW CREATE DATABASE CodeRadar;

-- 如果不是 utf8mb4，重新创建数据库
DROP DATABASE CodeRadar;
CREATE DATABASE CodeRadar DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```
